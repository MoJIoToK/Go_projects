Нормализация лимитов

## Описание

Есть входной файл с записями лимитов(остатки денег и бумаг для торговой системы на разные периоды) в текстовом формате.

Необходимо реализовать сервис на Go, который обрабатывает эти данные по заданным правилам, сохраняет результат в базу
данных и предоставляет HTTP API для запуска обработки и просмотра результата.

---

Лимиты по торговым инструментам имеют вид:

DEPO: FIRM_ID = #; SECCODE = #; CLIENT_CODE = #; LIMIT_KIND = #; OPEN_BALANCE = #.#; OPEN_LIMIT = #.#; TRDACCID = #;
WA_POSITION_PRICE = #.#;

Лимиты по денежным средствам имеют вид:

MONEY: FIRM_ID = #; TAG = #; CURR_CODE = #; CLIENT_CODE = #; LIMIT_KIND = #; OPEN_BALANCE = #.#; OPEN_LIMIT = #.#;
LEVERAGE = #.#;

---

Позиции по торговым инструментам, которые содержат нулевые остатки во всех видах лимита т.е.: LIMIT_KIND ∈ {0, 1, 2,
365} и OPEN_LIMIT = 0 - необходимо удалить.

При этом, если у клиента совсем не останется позиций по торговым инструментам - необходимо нарисовать или оставить один
лимит (OPEN_LIMIT) = 0 по любому существующему торговому инструменту (SECCODE) c LIMIT_KIND = 0.

Где позиция по инструменту (SECCODE) != 0 проверить существование всех лимитов (LIMIT_KIND) их должно быть 4: {0, 1, 2,
365}, иначе сообщить об ошибке, но обработку продолжаем - клиента оставляем с теми лимитами, которые есть в файле.

---

## Хранение данных

Результат обработки должен сохраняться в реляционную базу данных.

---

## HTTP API

### POST /limits/normalize

Назначение:

- принять входные данные (текст или файл);

- выполнить обработку по описанным правилам;

- сохранить результат в базу данных.

- иметь возможность выгрузить данные.
